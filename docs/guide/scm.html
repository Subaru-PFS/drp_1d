I. Introduction:

The code base of all the software is stored in a git repository.
The currently used git repo is located at:
  • http://gitlab.oamp.dev/ez_analysis/ez_rework
Management rules for commit and branching follow the rules described in the following document:
  • A successful branching model: http://nvie.com/posts/a-successful-git-branching-model/ 

II. Branch description:

1. Master branch:

Master branch on GitLab: http://gitlab.oamp.dev/ez_analysis/ez_rework/tree/master/

The master branch is the branch where you can get the latest released of the source code.
No work happen on this branch.
Code from develop branch get merged into master branch when a new release needs to be delivered.
For each release a new tag is created following those conventions (described here):

Tag name = Major.Minor.Patch

Where:
    • MAJOR version when you make incompatible API changes,
    • MINOR version when you add functionality in a backwards-compatible manner, and
    • PATCH version when you make backwards-compatible bug fixes.
    - How to create a feature branch from develop:

    • 2. Develop branch:

    Develop branch on GitLab: http://gitlab.oamp.dev/ez_analysis/ez_rework/tree/develop/

    The develop branch is the branch where you can get the latest development source code.
    Most of the time, o work happen on this branch.
    Code from features branch get merged into develop branch each time a feature or a fix is completed on it's own separated feature branch.

    - How to re integrate a feature branch back to develop:

    cd $ROOT_DIR
    git checkout develop
    git merge --no-ff feature_branch_name
    git branch -d feature_branch_name
    git push origin develop

    3. "Feature" branch:

    Feature branch is a generic name for each branch forked FROM the develop branch and used as an intermediary working area for features and bug fix development.
    As soon as the developement of the feature or bug fix is finished, the branch must be re integrated back into develop.
    Note: re-integration on the develop branch MUST BE DONE with the --no-ff flags in order to keep feature branch separted from develop

    - How to create a feature branch from develop:

    cd $ROOT_DIR
    git checkout -b feature_branch_name develop
